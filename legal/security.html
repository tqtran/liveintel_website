Security at LiveIntel

Security is not a feature we add on top — it is the foundation everything else is built on. This page describes how we protect our platform, your data, and how to report vulnerabilities responsibly.

Security Practices

Infrastructure Security

  • All data in transit is encrypted with TLS 1.3; older protocol versions are not accepted.
  • All data at rest is encrypted with AES-256-GCM, including backups.
  • Infrastructure is hosted on SOC 2 Type II certified cloud providers with private networking and restricted egress.
  • Secrets and credentials are managed via a dedicated secret store; no credentials in source code or configuration files.
  • Production access is limited to a small number of authorised engineers using MFA and short-lived SSH certificates.
  • All infrastructure changes are applied through automated pipelines with audit trails.

Application Security

  • Code changes undergo mandatory peer review and automated static analysis (SAST) before merge.
  • Dependencies are monitored for known vulnerabilities using automated advisory scanning.
  • Regular penetration testing is conducted by independent third-party security firms.
  • HTTP security headers are enforced on all endpoints: HSTS, CSP, X-Frame-Options, and more.
  • Input validation and parameterised queries are enforced throughout the application stack.
  • Rate limiting and brute-force detection are applied to all authentication endpoints.

Access Control

  • Role-based access control (RBAC) is enforced at every API endpoint.
  • Principle of least privilege — users and services are granted only the minimum permissions required.
  • Multi-factor authentication (MFA) is available for all user accounts and required for all internal staff.
  • SAML 2.0 and OIDC are supported for enterprise single sign-on.
  • Session tokens are short-lived and automatically rotated.

Monitoring and Incident Response

  • All platform activity is logged to an immutable, append-only audit log.
  • Real-time security monitoring and anomaly detection are operational 24/7.
  • A formal incident response plan is in place with defined escalation paths and communication procedures.
  • Affected customers will be notified of any data breach within 72 hours of our becoming aware, in line with applicable law.

Vulnerability Disclosure Program

We welcome responsible disclosure from the security community. If you believe you have discovered a security vulnerability in LiveIntel, please report it to us before disclosing it publicly.

disclosure-process
Step 1Email a detailed report to security@liveintel.com
Step 2We acknowledge receipt within 48 hours
Step 3We investigate and confirm the issue within 5 business days
Step 4We develop and deploy a fix, keeping you informed
Step 5We coordinate public disclosure timing with you
RecognitionValid reports are credited in our security acknowledgements

Please include in your report: a description of the vulnerability, the affected component or URL, steps to reproduce, and potential impact. We ask that you act in good faith, avoid accessing or modifying user data, and refrain from denial-of-service testing.

We commit to responding promptly and working with you in good faith. Researchers who follow these guidelines will not face legal action from LiveIntel.
